Editing or Deleting a Code Mapping
If you determine that a source code has been mapped to the wrong code in the target system, you can correct it by editing the code mapping. Or if a code mapping needs to be deleted for any reason, you can delete it.
1. Select System Mapping from the sidebar on the left.
2. Identify the system mapping for which you want to make a code mapping correction, and click on the link in the Name column.
The System Mappings Details screen opens. The existing code mappings are listed in the Code Mappings table at the bottom of the screen.
3. Find the mapping you want to edit or delete by either scrolling through the list of code mappings, or by using the Filters option to search for it (see Filtering the Data on a Screen).
-To delete a code mapping: Click the Delete icon next to the desired code mapping.
-To edit a code mapping: Click the Edit icon next to the desired code mapping. The Edit Code Mapping drawer opens on the right. Make corrections to the fields below as necessary and then click SAVE. See Step 4 under Adding a Code Mapping for more information about how to select values for these fields.
□Source Code: Enter the code that is used in the source system.
□Target Code: Enter the equivalent code, in the target system, for the source code you just entered above.
