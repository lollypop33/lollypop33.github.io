Managing Codes
 
Adding a New Code via the Codes Option
Editing a Code via the Codes Option
Before you can map codes from a source system to a target system, the codes themselves must exist in the Data Management Application. You can create new codes in the app using either the Codes option or the System Mapping option. Here are the differences between the two options:
•During initial implementation, you might use the Codes option to quickly add all of the codes that are used in a particular system mapping, one after another. Then once you have added all of the codes, you can go into the System Mapping option, where you can specify which source code maps to which target code. See Adding a New Code via the Codes Option.
•After initial implementation, you might occasionally need to add a new code mapping to resolve a code equivalency error, which might require adding a new code. You can add a single new code on the fly, while in the process of adding a new code mapping. See Adding a Code Mapping.
