Message Statuses
An interface message has one of the statuses below. A messageâ€™s status is displayed in the Messages option, as well as on the Message Details screen.
Message Status
Description
Queued
The message is currently waiting to be processed. This is a transitory state.
Processing
The message is being processed now. This is a transitory state.
Failed
The message was not processed due to one or more errors. A message in this status will have at least one error with a severity of Failure, and possibly also additional errors with a severity of Warning.
Success
The message was processed successfully. A message in this status will not have any errors with a severity of Failure, but could have errors with a severity of Warning. An error with a severity of Warning does not prevent the message from being processed successfully.
Reprocessing
The message is being reprocessed now after a user addressed errors with the message and then chose to reprocess it (see Reprocessing Interface Messages). This is a transitory state.
Ignored
The message was received but cannot be processed because it does not match specific filter criteria to continue processing (for example, the interface criteria may filter out messages from a specific facility or messages of a specific type).
An interface message starts with a status of Queued and travels through the statuses as illustrated below.
