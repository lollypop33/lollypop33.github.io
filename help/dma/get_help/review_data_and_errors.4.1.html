Viewing Errors
The Data Management Application provides views into two types of integration information: interface messages and errors.
•The Messages option displays a list of all interface messages. From this option you can drill down to see the details of individual messages, and also view the errors that have occurred with a particular message (if any). See Viewing Messages.
•The Error Summary option displays a list of errors that occurred during the processing of the interface messages.This report groups together all the active errors that have the same error text, and displays the total number of messages with that error. From this option, you can drill down to see the details of a given error, and also view all of the specific interface messages that currently have that error. See Viewing Errors.
A given interface message can have more than one error, and can therefore be listed under multiple errors on the Error Summary.
EXAMPLE:
An interface message with ID “X1234” has two errors: “no matching code for IMG55” and “no matching code for ABN12.”
•In the Messages option, if you clicked on the message “X12345” to view its details, you would see both errors listed on the Message Details screen.
•In the Error Summary option, each error would be listed as a separate row on the screen. If you clicked on the first error for “no matching code for IMG55” to view its details, you would see message “X12345” listed on the Error Details screen (along with any other interface messages that had the same error). And if you clicked on the second error for “no matching code for ABN12” to view its details, you would also see the message “X12345” listed on this Error Details screen (along with other messages that had the same error).”
