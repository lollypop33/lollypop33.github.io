Viewing Errors
 
Viewing Error Details
Select the Error Summary option from the sidebar on the left to view a quick synopsis of all interface errors that are currently unresolved. This report is meant to be used as a work list of active issues that need to be addressed - as you resolve each error or mark it to be ignored, it is removed from this report.
The Error Summary groups together all errors that have the same error text. Each error is listed as an individual row on the report, and includes the following information:
•Error Text describing the error in limited detail.
•Source system from which the message originated.
•Target system to which the message was sent, or was intended to be sent.
•Error Category of the error (see Error Descriptions for a list of error categories).
•Severity of the error (see Error Statuses and Severity Levels for definitions of severity).
•Unresolved Count: The number of interface messages that have that error and that are still unresolved.
See Error Descriptions for a complete list of the various errors that might be displayed in this option.
The Error Summary is sorted in reverse chronological order, so that the most recent errors are listed at the top of the report. You can apply a filter, or a filter set, to narrow the report down to a particular subset of errors that you want to investigate (see Filtering the Data on a Screen).
Click the link in the Error Text column to see the details of an error (see Viewing Error Details).
