Resolving Code Equivalency Issues
When a code from the source system is not mapped to a code in the target system, an error will occur during the interface processing, and a “Code Lookup - No Match” error will display on the Error Summary. These errors can be resolved directly in the Data Management Application using the following process:
1. Add a new code mapping to map the source code to the correct target code. See Adding a Code Mapping.
2. Reprocess the interface messages that had the code equivalency error. See Reprocessing Interface Messages.
Another common code equivalency problem is when a code from the source system is mapped to the wrong code in the target system. This type of issue will only be identified if a user notices that the patient data in the source and target system do not match. It will not display as an error in the Data Management Application, but it could be resolved using the following process:
1. Change the mapping of the source code to the correct target code. See Editing or Deleting a Code Mapping.
2. In the Messages option, you could search for the interface messages that used the incorrect mapping and then reprocess them. See Reprocessing Interface Messages.
CAUTION: Please use care when determining whether to reprocess a message. Reprocessing runs the risk of sending an older version of data that could overwrite a newer version of that same data. For example, let’s assume that a message containing a preliminary radiology result failed. Then later, a different message with the final results for that same radiology result was successful. If you reprocessed the earlier message with the preliminary result, that preliminary result could replace the final result in the target system.
