Managing System Mappings
 
Adding a System Mapping
Editing a System Mapping
For a given data integration, use the System Mappings option to specify all of the code sets that need to be mapped between the source and target systems. The number of system mappings that are required for a data integration depends on the type of interface and how it is designed and implemented.
EXAMPLE: One-way Interface
For a one-way interface from System A to System B, you would typically create system mappings for each code set that is used by the interface. For example, you might create these system mappings:

Each interface is unique and may have different requirements. Consult with your Commure representative before creating system mappings. This is typically a one-time set-up that is performed at the initial implementation of a data integration within the Data Management Application.
