Resolving Errors
 
Marking Errors as Ignored

Reprocessing Interface Messages
After reviewing errors in the Error Summary or Messages options, you must determine a course of action.
•The error might be an issue that can only be resolved by making a change in the sending or receiving system, or via some other action taken outside of the Data Management Application. If this is the case, after resolving the issue, you might want to indicate that the error can now be ignored, so that it will fall off of the Error Summary report. See Marking Errors as Ignored.
•If the error is due to an issue with code equivalencies, then you might need to create, edit, or delete a code mapping. See Adding a Code Mapping or Editing or Deleting a Code Mapping.
See Error Descriptions for a complete list of the various errors that might be displayed in the Data Management Application.
