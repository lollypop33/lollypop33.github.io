Viewing Message Details
To view the details of a specific message:
1. Select Messages from the sidebar on the left.
2. Identify the message you want to view, and click the link in the Message ID column.
The Message Details screen opens and displays all of the information for that particular message.
At the top of the screen the following information is displayed:
•Message ID of the message.
•Source system from which the message originated.
•Target system to which the message was sent, or was intended to be sent.
•Status of the message (see Message Statuses for definitions of statuses).
•Received On date for the message. This is the date that the message was originally received and that the first attempt at processing it occurred. If a message fails processing, then there might be multiple attempts to process it; each attempt is a transaction.
•Last Processed On date for the message. This is the last time that any processing was done on this message, or the date of the last transaction for the message.
•Patient Name on the message.
•Patient MRN on the message.
At the bottom of the screen there are tabs for two tables: Errors and Transactions.
•Errors table: If the interface message has any errors, those errors are listed in an Errors table at the bottom of the screen. The table includes the following information for each error:
-Source system from which the message originated.
-Target system to which the message was sent, or was intended to be sent.
-Error Category of the error (see Error Descriptions for a list of error categories).
-Error Text describing the error in limited detail.
-Status of the error (see Error Statuses and Severity Levels for definitions of status).
-Severity of the error (see Error Statuses and Severity Levels for definitions of severity).
Please note that the Errors table shows only the errors from the latest attempt at processing the message.
EXAMPLE:
A message failed processing due to a “no matching code” error, which was displayed in the Errors table on the Message Details screen. If you added a new code mapping to correct the issue, and then reprocessed the interface message, the Errors table for that message would be updated to show information from this second processing only.
•If the message processed successfully without any errors, then the Errors table would be blank.
•If the second processing had errors, then the Errors table would display those new errors.
•Transactions table: This table shows all the transactions, or attempts at processing the message. If the message was successful the first time, then the Transactions table contains only one row. If the message failed to process one or more times, then all of the attempts/transactions are listed here. The table includes the following information for each transaction:
-Transaction ID of the message transaction. If you click the Transaction ID link, then the following information is displayed:
□Original Message: The full text of the original interface message.
□Transformed Message: The full text of the message as it was transformed for the target system. This information may not be present for messages that failed processing, depending on the nature of the error that caused the failure.
-Date/Time of the message transaction.
-Status of the message transaction (see Message Statuses).
-Source system from which the message originated.
-Target system to which the message was sent, or was intended to be sent.
