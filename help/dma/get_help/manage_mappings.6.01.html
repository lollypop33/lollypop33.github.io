Managing Code Mappings
In order for an interface from one system to another to work, any code sets used by those two systems must be mapped. For example, if you are interfacing Radiology data from one system to another, then typically each system will have code sets for items such as Tests, Results, Facilities, and so on. This means that if, for example, a Test code exists in the source system as IMG123, which represents an “x-ray of the mandible,” then it must be mapped to whatever is the equivalent code in the target system.
Depending on the specifications of the interface, in some cases all of the codes in a code set must be mapped, even if the source and target systems both use the same code for the same item. In other cases, you may be required to map only those codes where the source and target system use a different code for the same item.
The steps to configure system and code mappings are as follows:
1. For each interface, create a system mapping for each of the necessary code sets. See Managing System Mappings.
2. For each source and target system within an interface, create all the codes that will require mapping. See Managing Codes.
3. For each system mapping, map the codes from the source system to the equivalent codes in the target system. See Managing Code Mappings.
