Error Descriptions
The following table describes the error categories and error messages that you might encounter in the Data Management Application. Severity levels are described in Error Statuses and Severity Levels.
Error Category
Sample Error Messages
Severity Level/Explanation
Code Lookup - No Match
No matching code for [code] of [field name] found in the [system name].
WARNING or FAILURE
For a non-required field (WARNING) or a required field (FAILURE), a code from the source system is not mapped to a code in the target system. (A new code mapping should be added to map the source code to the correct target code.)
Destination Unreachable
The processing is complete but the destination is unreachable.
CRITICAL
Processing of the message is complete, but Transporter cannot send the message to the destination. This may be due to a simple network outage, or it could occur if the destination IP address is incorrect. Requires intervention by Commure support.
Invalid Message Received
Invalid message received in the following location [location name]
Multiple repetition received for OBR-4 segment. Only one repetition for OBR-4 segment is permitted.
No OBR segment received, expected 1 OBR segment.
No PID segment in the message.
No PV1 segment in the message.
Missing code system in the OBR-4 segment.
Missing MRN in PID-3 segment.
Missing identifier in PID-18.
Missing value for OBX-8 segment.
FAILURE
The message has a structural problem (does not match the interface specifications) and cannot be processed.
NOTE: The errors in this category can vary by client since they are based on the specifications of the data integration.
Transform Error
The transform experienced unexpected error.
FAILURE
There is a bug in the transformation of the message. It may be an actual bug, or it may be a deficiency in the transformation specifications.
Unexpected Error
Unexpected error happened and please contact Commure Support!
CRITICAL or UNDEFINED
There is a critical problem between Transporter and the server storing code equivalencies. Depending on the nature of the issue, this error can have a severity of CRITICAL or UNDEFINED. Requires intervention by Commure support.
For example:
•Posting GraphQL throws an HTTP 500 error
•Pod is on a continuously crashing loop
