Reprocessing Interface Messages
 
Reprocessing Messages from the Error Summary Option
Reprocessing Messages from the Messages Option
You can reprocess one or more interface messages after errors or issues with the messages have been resolved. Reprocessing a message sends it through the interface engine again, so that it can be processed correctly and reach the target system.
CAUTION: Please use care when determining whether to reprocess a message. Reprocessing runs the risk of sending an older version of data that could overwrite a newer version of that same data. For example, let’s assume that a message containing a preliminary radiology result failed. Then later, a different message with the final results for that same radiology result was successful. If you reprocessed the earlier message with the preliminary result, that preliminary result could replace the final result in the target system.
There are two approaches to reprocessing a message:
•From the Error Summary option, select an error and then reprocess one or more of the messages that had that error. See Reprocessing Messages from the Error Summary Option.
•From the Messages option, search for a specific message or set of messages, and then reprocess one or more of them. See Reprocessing Messages from the Messages Option.
